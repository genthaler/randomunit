21  /**
22   * This class facilitates the creation of randomized tests. Deterministic,
23   * hand-written tests usually cover only limited amount of cases, and can miss
24   * subtle bugs. Randomized tests can help in this regard, as they can cover a
25   * great deal of cases automatically. Still, it is sensible to write specific
26   * (usual) tests for rare corner cases, to complement the randomized search.
27   * <p>
28   * To implement a randomized test, we need some collaborating objects. These
29   * objects are grouped into named <i>pools</i>, that denote objects with similar
30   * semantics. For instance, an object pool named "keys" could be containing
31   * objects that are used as keys in some map. By effectively organizing the
32   * objects into groups, the search space is very much smaller, so the test
33   * concentrate more on cases that actually matter.
34   * <p>
35   * The creation of a randomized test involves defining a subclass of this class,
36   * and defining object pools and random test methods. Both these goals are
37   * reached by simply annotating the test methods. An example follows, that
38   * depicts a method that tests a special stack implementation that does not
39   * accept null values.<BR>
40   * <BR>
41   * <p>
42   * 
43   * <pre>
44   * &#064;Prob(0.5)
45   * &#064;Params({ &quot;stackPool&quot;, &quot;elementPool&quot; })
46   * public void check(MyStack stack, Object element) throws Exception {
47   * 	precondition(element != null, &quot;Null elements are not allowed&quot;);
48   * 	precondition(!stack.isFull(), &quot;Stack cannot be empty&quot;);
49   * 
50   * 	int size = stack.size();
51   * 	stack.push(element);
52   * 
53   * 	postcondition(!stack.isEmpty(), &quot;Stack is not empty&quot;);
54   * 	postcondition(stack.size() == size + 1, &quot;Stack size grows by one&quot;);
55   * }
56   * </pre>
57   * <p>
58   * The &#064;Params annotation specifies the names of the respective object
59   * pools, from which to choose random members to inject in the parameters of the
60   * method. This concept is a form of dependency-injection: the method describes
61   * its dependencies, and the "container" (the randomized test) injects
62   * appropriate values. Note that &#064;Params is necessary as the type of the
63   * arguments cannot express possible semantics differences between objects of
64   * the same type (ie, objects of same class that are used in a substantially
65   * different way). Please note that the methods are not forced to specify the
66   * exact type of the arguments: any super type will do. Also, primitive
67   * parameters can be defined. Yet, if a primitive argument is prescribed, and
68   * the specified object pool contains null values, there is the possibility that
69   * <code>null</code> will be randomly selected and attempted to be injected as a
70   * primitive, but this is impossible and an exception will be thrown.
71   * <p>
72   * A random test method usually is composed by three parts: preconditions,
73   * actual testing of some object via its methods, and postconditions. If a
74   * precondition fails, nothing happens; this just means that the random
75   * arguments that were chosen were illegal for the respective action to take
76   * place. This is not counted as a complete step. But if the testing of the
77   * object throws an exception, this constitutes a bug: valid arguments
78   * (preconditions passed) caused a failed execution. If any postcondition fails,
79   * this is also a bug: the object under test did not maintain its contract.
80   * <p>
81   * How are objects inserted into object pools? By the means of random test
82   * methods too. Each method that has a &#064;Prob annotation (i.e., a random
83   * test method), can return a value and request it to be added to a object pool
84   * (or more), as declared by the &#064;Creates annotation. This is done like
85   * this:
86   * <p>
87   * 
88   * <pre>
89   * &#064;Creates(&quot;elementPool&quot;)
90   * &#064;Prob(0.8)
91   * public int createRandomElement() {
92   * 	return random.nextInt(10); // this is added into the &quot;elementPool&quot; pool as
93   * 								// Integer automatically
94   * }
95   * </pre>
96   * <p>
97   * A method can very well play the two roles simultaneously. It can declare a
98   * &#064;Creates annotation and return a value, yet it can take arguments and
99   * declare a &#064;Params annotation. This notion captures the expressiveness
100  * needed for cases where we specifically want to test objects that are derived
101  * from (or anyhow depended on) other objects into consideration.
102  * <p>
103  * Random test methods do not have to start with "test" prefix. Such methods
104  * will be considered by JUnit as usual. If wanted, some prefix like "random"
105  * can be used for method names, but this is not required.
106  * <p>
107  * When a random method is chosen, that needs objects from pools that are
108  * currently empty, the method is skipped, and it is not recorded as an executed
109  * step. So, if there is only one test method that requires a parameter, and the
110  * only method that creates an object for the respective object pool is never
111  * called (for instance, it has a zero probability), an infinite loop will
112  * ensue.
113  * <p>
114  * What if we want to create many objects at the start, and after that only a
115  * few? This case is satisfied by using different <i>phases</i> during the test.
116  * Each &#064;Prob actually declares an array of doubles: the first double is
117  * the method's relative probability for phase 0, the second for phase 2 etc, as
118  * in this example:
119  * <p>
120  * 
121  * <pre>
122  * &#064;Prob({2, 0}
123  * void myMethod(...) { }
124  * </pre>
125  * 
126  * The example shows a method that will never be executed at the second phase
127  * (phase == 1), but it has a positive probability ( == 2) in the first phase
128  * (phase == 0). Trailing zero probabilities are implied. The randomized test
129  * can change the current phase wherever it wants. A good place is at the <a
130  * href="#onStep(int)">onStep(int executedSteps)</a> call-back method, that is
131  * called after the execution of each random test method.
132  * <p>
133  * Invariant checking is also supported. Annotate a method with
134  * &#064;Invariant("poolName"), that tests the invariant of some object
135  * belonging to the specified object pool. Such a method must have exactly one
136  * argument defined, that is injected by a value from the respective object
137  * pool. Here is an example:
138  * <p>
139  * 
140  * <pre>
141  * &#064;Invariant(&quot;stackPool&quot;)
142  * public void checkStackInvariant(Stack stack) {
143  * 	invariant(stack.isEmpty() &circ; stack.size() &gt; 0); // stack is empty XOR stack
144  * 													// size is not zero, not
145  * 													// both
146  * }
147  * </pre>
148  * <p>
149  * An object pool can have as many invariant-testing methods associated with it
150  * as desired. After each random method invocation, <strong>all</strong>
151  * respective invariant-testing methods are called for every argument that was
152  * supplied to the method. If the method is declared as &#064;Creates, the
153  * respective invariant-testing methods are called for its return value as well.
154  * <p>
155  * For further customization, another call-back method is provided, <a
156  * href="#filterNewObject(java.lang.String, java.lang.Object)">
157  * filterNewObject(String poolName, Object newObject)</a> which can be overrided
158  * to control exactly the objects that are inserted into an object pool.
159  * <p>
160  * A LogStrategy is associated with each RandomizedTestCase. It logs method
161  * invocations, along with information about its parameters and the pools from
162  * which they were chosen.
163  * <p>
164  * Finally, the contents of the object pools can be always be queried using the
165  * method <a href="#getPool(String)> getPool(String poolName)</a>.
166  * 
167  * 
168  * @author Andreou Dimitris, andreou &lt at &gt csd dot uoc dot gr
169  */

193 	/**
194 	 * Creates a randomized test that will execute as many random steps as
195 	 * specified, and will log a specified amount of steps in an internal cyclic
196 	 * buffer.
197 	 * 
198 	 * To successfully call this constructor, a list of rules apply: - Every
199 	 * method that declares a &#064;Creates annotation, cannot have void return
200 	 * type. - Every method that declares a &#064;Prob annotation must be
201 	 * non-private. - Every method that declares a &#064;Prob annotation and has
202 	 * parameters, it must declare a &#064;Params annotation with a valid pool
203 	 * name for each parameter. - At least one &#064;Creates method must exist.
204 	 * - Each &#064;Invariant method must have exactly one parameter. - No
205 	 * method can both declare &#064;Creates and &#064;Invariant. -
206 	 * &#064;Invariant annotations must specify an object pool which at least
207 	 * one &#064;Creates specifies. - At least one random test method
208 	 * (&#064;Prob) must be declared.
209 	 * 
210 	 * 
211 	 * @param testName
212 	 *            the name of the test (used for JUnit)
213 	 * @param steps
214 	 *            the number of random steps to execute (must be > 0)
215 	 * @param logStrategy
216 	 *            the strategy that will handle the produced log entries. May be
217 	 *            null (a default LogStrategy will be used)
218 	 * @see #getLogStrategy()
219 	 */
220 	public RandomizedTestCase(String testName, int steps,
221 			LogStrategy logStrategy) {
222 		super(testName);
223 		if (testName.startsWith("test") && testName.length() > 4) {
224 			// ignore testXXX methods, but do not ignore test() method used by
225 			// this class
226 			// The constructor was called by JUnit to execute a test method - do
227 			// nothing
228 			this.steps = 0;
229 			this.logStrategy = null;
230 			this.probManagers = null;
231 			return;
232 		}
233 		if (steps <= 0) {
234 			throw new IllegalArgumentException("Steps must be > 0");
235 		}
236 		if (logStrategy == null) {
237 			logStrategy = new SimpleLogStrategy(0);
238 		}
239 		this.logStrategy = logStrategy;
240 		this.steps = steps;
241 		initPools();
242 		initInvariants();
243 		probManagers = initMethods();
244 	}

246 	private void initPools() {
247 		for (Method m : getClass().getDeclaredMethods()) {
248 			Creates creator = m.getAnnotation(Creates.class);
249 			if (creator == null) {
250 				continue;
251 			}
252 			if (m.getReturnType().getName().equals("void")) {
253 				throw new RuntimeException(
254 						"@Creator annotation cannot be declared in a method that has void return type: "
255 								+ m);
256 			}
257 			for (String poolName : creator.value()) {
258 				if (!pools.containsKey(poolName)) {
259 					pools.put(poolName, new ArrayList<Object>());
260 					invariants.put(poolName, new ArrayList<Method>());
261 				}
262 			}
263 		}
264 	}
265 
266 	private void initInvariants() {
267 		for (Method m : getClass().getDeclaredMethods()) {
268 			Invariant invariant = m.getAnnotation(Invariant.class);
269 			if (invariant == null) {
270 				continue;
271 			}
272 			if (m.getAnnotation(Creates.class) != null) {
273 				throw new RuntimeException(
274 						"Illegal combination of annotations: @Invariant cannot coexist with @Creator, in method: "
275 								+ m);
276 			}
277 			if (m.getParameterTypes().length != 1) {
278 				throw new RuntimeException(
279 						"Method: '"
280 								+ m
281 								+ "' was annotated with @Invariant, so it MUST take exactly one parameter");
282 			}
283 			if (Modifier.isPrivate(m.getModifiers())) {
284 				throw new RuntimeException(
285 						"Method: '"
286 								+ m
287 								+ "' has an @Invariant annotation - must not be private");
288 			}
289 			List<Method> methods = invariants.get(invariant.value());
290 			if (methods == null) {
291 				throw new RuntimeException("Method: '" + m
292 						+ "' has an @Invariant annotation which has "
293 						+ "an invalid object pool name: '" + invariant.value()
294 						+ "'");
295 			}
296 			methods.add(m);
297 			m.setAccessible(true);
298 		}
299 	}

301 	private List<ProbabilitiesManager> initMethods() {
302 		List<ProbabilitiesManager> managers = new ArrayList<ProbabilitiesManager>();
303 		int count = 0;
304 		for (Method m : getClass().getDeclaredMethods()) {
305 			Prob prob = m.getAnnotation(Prob.class);
306 			if (prob == null) {
307 				continue;
308 			}
309 			if (Modifier.isPrivate(m.getModifiers())) {
310 				throw new RuntimeException("Method: '" + m
311 						+ "' has a @Prob annotation - must not be private");
312 			}
313 			if (m.getParameterTypes().length > 0) {
314 				Params params = m.getAnnotation(Params.class);
315 				if (params == null) {
316 					throw new RuntimeException(
317 							"Misconfigured method declaration: Method '"
318 									+ m
319 									+ "' has "
320 									+ "a @Prob annotation, accepts arguments, but has not @Params annotation. "
321 									+ "Declare a @Params annotation that defines a name of an object pool from which"
322 									+ " to inject the parameters for the method");
323 				}
324 				if (params.value().length != m.getParameterTypes().length) {
325 					throw new RuntimeException(
326 							"Illegal number of elements in Params annotation; was "
327 									+ params.value().length + ", method needs "
328 									+ m.getParameterTypes().length
329 									+ " parameters. Method signature=" + m);
330 				}
331 				for (String param : params.value()) {
332 					if (!pools.containsKey(param)) {
333 						throw new RuntimeException(
334 								"Undefined pool name in Params annotation: '"
335 										+ param + "', "
336 										+ "declared in method: " + m);
337 					}
338 				}
339 			}
340 			count++;
341 			double[] probs = prob.value();
342 			for (int i = 0; i < probs.length; i++) {
343 				if (managers.size() <= i) {
344 					managers.add(new ProbabilitiesManager());
345 				}
346 				ProbabilitiesManager manager = managers.get(i);
347 				manager.addMethod(m, probs[i]);
348 			}
349 		}
350 		if (count == 0) {
351 			throw new AssertionFailedError("No test method found");
352 		}
353 		return managers;
354 	}

364 	/**
365 	 * This method is called by JUnit and fires the randomized test.
366 	 */
367 	public final void test() {
368 		try {
369 			executedSteps = 0;
370 			while (executedSteps < steps) {
371 				if (invokeMethod()) {
372 					executedSteps++;
373 					onStep(executedSteps);
374 				}
375 			}
376 		} catch (TestFailedException e) {
377 			examineException(e);
378 			throw e;
379 		}
380 	}

393 	private boolean invokeMethod() {
394 		ProbabilitiesManager manager = probManagers.get(currentPhase);
395 		Method m = manager.chooseMethod(random);
396 		if (m == null) {
397 			if (m == null) {
398 				throw new RuntimeException(
399 						"All probabilities are zero, currentPhase="
400 								+ currentPhase);
401 			}
402 		}
403 		Class[] types = m.getParameterTypes();
404 		Object[] args = new Object[0];
405 		Params params = null;
406 		if (types.length > 0) {
407 			params = m.getAnnotation(Params.class); // guaranteed to return
408 													// non-null
409 			String[] poolNames = params.value();
410 			args = new Object[poolNames.length];
411 			for (int i = 0; i < args.length; i++) {
412 				List<Object> pool = pools.get(poolNames[i]); // guaranteed to
413 																// return
414 																// non-null
415 				if (pool.isEmpty()) {
416 					return false;
417 				}
418 				args[i] = pool.get(random.nextInt(pool.size()));
419 			}
420 		}
421 		try {
422 			Object o = m.invoke(this, args);
423 			Creates creator = m.getAnnotation(Creates.class);
424 			if (creator != null) {
425 				try {
426 					for (String poolName : creator.value()) {
427 						pools.get(poolName).add(filterNewObject(poolName, o));
428 					}
429 				} catch (PreconditionFailedException e) {
430 					return false;
431 				}
432 			}
433 			if (args.length > 0) { // check invariants
434 				for (int i = 0; i < args.length; i++) {
435 					if (args[i] == null) {
436 						continue; // ignore invariants for null
437 					}
438 					String type = params.value()[i];
439 					checkInvariants(args[i], type);
440 				}
441 			}
442 			if (o != null && creator != null) {
443 				for (String poolName : creator.value()) {
444 					checkInvariants(o, poolName);
445 				}
446 			}
447 			PooledObject[] pooledArgs = createPooledObjects(args,
448 					params != null ? params.value() : new String[0]);
449 			String[] pools = creator != null ? creator.value() : new String[0];
450 			logStrategy.appendLog(new MethodInvocationLog(m, pooledArgs, o,
451 					pools));
452 		} catch (IllegalAccessException e) {
453 			throw new Error(
454 					"Could not invoke method - maybe a null was attempted to be used as a primitive?",
455 					e); // will never happen - already checked that method is
456 						// public
457 		} catch (InvocationTargetException e) {
458 			Throwable cause = e.getCause();
459 			if (cause instanceof PreconditionFailedException) {
460 				// ignore failed preconditions
461 				return false;
462 			}
463 			MethodInvocationLog methodLog = new MethodInvocationLog(m,
464 					createPooledObjects(args, params != null ? params.value()
465 							: new String[0]), null, new String[0]);
466 			try {
467 				throw e.getCause();
468 			} catch (PostconditionFailedException post) {
469 				// this is a bug
470 				throw new TestFailedException(
471 						"Failed postcondition while invoking method: '" + m
472 								+ "' " + "with args: " + Arrays.asList(args),
473 						methodLog, post, logStrategy);
474 			} catch (InvariantFailedException invEx) {
475 				// user called invariant() or threw manually an
476 				// InvariantFailedException
477 				throw new TestFailedException(
478 						"Failed invariant while invoking method: '" + m + "' "
479 								+ "with args: " + Arrays.asList(args),
480 						methodLog, invEx, logStrategy);
481 			} catch (TestFailedException invEx) {
482 				// user called checkInvariants
483 				throw invEx;
484 			} catch (Throwable t) {
485 				// this is also a bug
486 				throw new TestFailedException(
487 						"Exception thrown while invoking method: '" + m + "' "
488 								+ "with args: " + Arrays.asList(args)
489 								+ ", for which" + " no precondition failed",
490 						methodLog, t, logStrategy);
491 			}
492 		}
493 		return true;
494 	}

496 	private PooledObject[] createPooledObjects(Object[] args, String[] poolNames) {
497 		PooledObject[] pooled = new PooledObject[args.length];
498 		for (int i = 0; i < args.length; i++) {
499 			pooled[i] = new PooledObject(args[i], poolNames[i]);
500 		}
501 		return pooled;
502 	}

504 	/**
505 	 * Runs all invariant testing methods (marked with &#064;Invariant) that are
506 	 * associated with a object pool, providing as argument to the testing
507 	 * methods the specified object. This method is called automatically after
508 	 * each random method execution, once for each method's parameters, and one
509 	 * time for each addition to an object pool of the returned value. That is,
510 	 * if a method is marked with &#064;Creates({"pool1", "pool2"}), the return
511 	 * value will be checked against the invariants of both pool1 and pool2.
512 	 * 
513 	 * Test implementations are free to call this method wherever it is
514 	 * appropriate, to test invariants for objects for which they are not tested
515 	 * automatically (i.e., if the objects were not injected as parameters or
516 	 * were not a return value, but were accessed with other means).
517 	 */
518 	protected final void checkInvariants(Object o, String poolName) {
519 		if (!invariants.containsKey(poolName)) {
520 			throw new IllegalArgumentException("Invalid pool name defined: '"
521 					+ poolName + "', legal values=" + invariants.keySet());
522 		}
523 		for (Method invMethod : invariants.get(poolName)) {
524 			List<Object> pool = pools.get(poolName); // guaranteed to return
525 														// non-null AND
526 														// non-empty
527 			try {
528 				invMethod.invoke(this, new Object[] { o });
529 			} catch (InvocationTargetException e) {
530 				MethodInvocationLog methodLog = new MethodInvocationLog(
531 						invMethod, new PooledObject[] { new PooledObject(o,
532 								poolName) }, null, new String[0]);
533 				try {
534 					throw e.getCause();
535 				} catch (InvariantFailedException ex) {
536 					throw new TestFailedException(
537 							"Failed invariant while invoking method: '"
538 									+ invMethod + "' " + "with argument: " + o,
539 							methodLog, ex, logStrategy);
540 				} catch (Throwable t) {
541 					throw new TestFailedException(
542 							"Invariant-checking method: '" + invMethod
543 									+ "' caused an error", methodLog, t,
544 							logStrategy);
545 				}
546 			} catch (IllegalAccessException e) {
547 				throw new Error(e); // will never happen
548 			}
549 		}
550 	}

707 	private static class ProbabilitiesManager {
708 		private double maxProb = 0.0;
709 		private final Map<Method, Double> probs = new LinkedHashMap<Method, Double>();
710 
711 		void addMethod(Method m, double prob) {
712 			m.setAccessible(true);
713 			if (m == null) {
714 				throw new IllegalArgumentException("Null method");
715 			}
716 			if (prob < 0.0) {
717 				throw new IllegalArgumentException("Illegal probability: "
718 						+ prob);
719 			}
720 			maxProb += prob;
721 			probs.put(m, prob);
722 		}
723 
724 		public Method chooseMethod(Random random) {
725 			double next = random.nextDouble() * maxProb;
726 			double curr = 0.0;
727 			Method m = null;
728 			for (Entry<Method, Double> entry : probs.entrySet()) {
729 				m = entry.getKey();
730 				double prob = entry.getValue();
731 				curr += prob;
732 				if (curr > next) {
733 					return m;
734 				}
735 			}
736 			return m;
737 		}
